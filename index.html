005 / Process

        
How I work as a design engineer.

      

      

        

          
→ 01

          
Design in the browser

          
Figma is for thinking. The browser is where ideas have to survive contact with real text, real data, and real touch targets. I do my hi-fi work in code so what I show is what ships.

          

            HTML
            CSS
            React
            Tailwind
          

        

        

          
→ 02

          
Prototype with Claude

          
Pair-design with an LLM to compress the cycle: sketch → prototype → critique → ship. The result is a clickable artifact stakeholders can actually use before a single engineer commits time.

          

            Claude
            JSX
            Live data
            Tweaks
          

        

        

          
→ 03

          
Hand off without throwing it

          
Real handoff is opening a PR. I write the tokens, primitives, and a few real screens — eng inherits a working artifact, not a Figma export and a prayer.

          

            PRs
            Tokens
            Storybook
            Trio sync
